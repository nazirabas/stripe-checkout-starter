# Contributing

Thanks for your interest. This is a small starter repo, so contributions are welcome but scoped tight.

## Ways to help

- **Bug reports** — open an issue with the smallest repro you can
- **Docs fixes** — typos, clearer examples, better ordering. Just open a PR
- **New examples** — additional usage patterns are welcome in `examples/` or the README
- **New features** — please open an issue first to discuss scope before writing code

## Pull request checklist

- Branch from `main`
- Keep the change focused — one feature or fix per PR
- Add or update relevant docs in the README
- Make sure `npm run build` passes locally (CI will check)
- Use conventional commit style if you can (`fix:`, `feat:`, `docs:`)
